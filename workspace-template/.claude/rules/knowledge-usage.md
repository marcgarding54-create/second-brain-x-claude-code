# Knowledge Base Protocol

## At the Start of a New Project/Task
1. Load `@knowledge-base/INDEX.md`
2. Check which topic files are relevant
3. Load ONLY the relevant files with @-syntax
4. Start the project with this context

## During Work
- Keep findings internally (no automatic writing)
- For solved problems: note internally for later export

## After Project Completion or on Request
Write the following to the appropriate knowledge files:
- Solved errors → `knowledge-base/[domain]/errors.md`
- Proven patterns → `knowledge-base/[domain]/patterns.md`
- Reusable configs → `knowledge-base/[domain]/templates.md`
- General learnings → `knowledge-base/general/lessons-learned.md`

## Keep INDEX.md Current
After every topic file update: update INDEX.md entry.
